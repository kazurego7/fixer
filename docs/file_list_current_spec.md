# ファイル一覧の現行仕様

2026-04-05 時点の `/files/` と `/files/view/` の現行仕様をまとめる。
ファイル一覧のリファクタリングや高速化では、この挙動を原則維持する。

## 一覧画面

- チャット画面の Git ステータスラインから `/files/` に遷移する。
- 一覧はツリー形式で表示する。
- ルート一覧は `/api/repos/file-tree` から取得する。
- フォルダを開いたときだけ、そのフォルダの子を `/api/repos/file-tree?path=...` で取得する。
- ただし `hasDiff=true` のフォルダは初期状態で開いており、表示に必要な子も自動で取得する。
- `hasDiff=false` のフォルダは初期状態で閉じている。
- ルート一覧の初回読み込み時だけ `ファイル一覧を読み込み中...` を表示する。
- フォルダ展開時の子取得では `読み込み中...` を表示しない。
- `diff なしも表示` が OFF のときは、差分ありファイルと、その表示に必要なフォルダだけを表示する。
- `diff なしも表示` が ON のときは、差分なしファイルと除外ファイルも表示する。
- 除外ファイルは通常の差分なしファイルと同様に、一覧では `+/-` を表示しない。
- フォルダ名とファイル名の色は Git 状態で変わる。
- フォルダ色は配下の最優先状態を引き継ぐ。

## 開閉ルール

- 差分ありファイルを含むフォルダは初期状態で開く。
- 差分なしのみのフォルダは初期状態で閉じる。
- 除外フォルダは初期状態で閉じる。
- ユーザーが開閉した状態は、その画面の state に保持する。
- `diff なしも表示` を切り替えるとツリー state は再初期化される。

## 詳細画面

- 一覧のファイル行を押すと `/files/view/?path=...` に遷移する。
- 詳細画面のタイトルはファイルの相対パスを表示する。
- 差分ありファイルは、単一のコード面の中で GitHub 風の unified diff 表示にする。
- 差分なしファイルは、本文だけを 1 列の行番号付きで表示する。
- 除外ファイルも差分なし扱いで、本文だけを表示する。
- 除外ファイルは新規ファイル風の `+` diff を生成しない。
- 詳細画面の状態チップは `変更 / 追加 / 削除 / 除外 / 差分なし` を表示する。

## 導線

- `前の diff` / `次の diff` は差分ありファイルだけを対象に移動する。
- チャット返答のローカルファイルリンクは `/files/view/` に遷移する。
- 行番号付きリンクは対象行へスクロールし、その行をハイライトする。
- ローカルファイルでないリンクは新しいタブで開く。
